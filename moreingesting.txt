❯ Can I ingest incrementally? LIke 2 chapters to today, 3 tomorrow and a    
  few more later? Index each time?                                          

● Let me check how the store factory initializes the FAISS store — specifically
  whether it loads an existing index.

● Read 1 file (ctrl+o to expand)

● Yes, incremental ingestion works correctly. The chain is:

  1. VectorStoreFactory creates FAISSVectorStore with persist_dir=".vectorstore"
  2. FAISSVectorStore.__init__() immediately calls _load_from_disk() — so it
  loads your existing index first
  3. add_embeddings() checks if a chunk ID already exists and updates in place,
  or adds new ones
  4. save() writes the merged result back to disk

  So your workflow can be either:

  Option A — point to only the new files each time:
  # Day 1
  python scripts/ingest.py --corpus ./data/batch1 --index

  # Day 2 — existing index loaded, new chunks appended
  python scripts/ingest.py --corpus ./data/batch2 --index

  Option B — just drop new files into ./data and re-run on the whole folder:
  python scripts/ingest.py --corpus ./data --index
  Chunks that already exist get updated in place (no duplication), new ones get
  added.

  Option B is simpler and safe to repeat as many times as you like.
